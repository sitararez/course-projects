Part C: Reflection

When looking at the two methods, each has its own benefits and drawbacks. CSS allows you to have complete control over every detail, letting you customize the layout, style, and responsiveness exactly how you want. This flexibility is great for creating a design that matches your vision. However, it can be time-consuming since it often involves manually adjusting elements, testing sizes, and tweaking breakpoints through trial and error. Media queries can be particularly challenging because you need to consider different screen sizes and devices. On the other hand, CSS provides a solid way to organize content, but you must build the entire layout from the ground up. I found CSS to be more intuitive since I had direct control, but it took a lot of time, especially to get the responsive features right.

In contrast, Bootstrap offers ready-made classes, grids, and tools that speed up the development process. Making a webpage responsive is easier with Bootstrap because its grid system is designed for various screen sizes, requiring less code. This helps create professional-looking layouts without needing to manually set the size or responsiveness of each element. The downside is that you may feel limited by the framework's choices. Customizing beyond Bootstrapâ€™s defaults can be time-consuming and may need some research into the documentation. Learning the grid system took some effort, but once I understood it, it saved a lot of time compared to writing CSS from scratch. Overall, I found Bootstrap to be faster and more efficient for styling and responsiveness. The convenience of not having to adjust every element for different screen sizes was a big plus. While CSS offered more control, I preferred the ease and speed of Bootstrap for this project, especially when considering time and functionality.
